# Ralph Progress Log

This file tracks progress across iterations. Agents update this file
after each iteration and it's included in prompts for context.

## Codebase Patterns (Study These First)

- **Rust toolchain**: Must use rustup stable (1.93+). Homebrew rustc is outdated (1.87). Ensure `$HOME/.cargo/bin` is first in PATH.
- **Workspace layout**: `engine/Cargo.toml` is the workspace root with `r2-core`, `r2-ffi`, `r2-cli` members. All share `version.workspace = true`.
- **Config pattern**: `Config::load()` returns `Config::default()` when file is missing. All fields use `#[serde(default)]` so partial TOML files work. `R2DROP_HOME` env var overrides `~/.r2drop/`.
- **FFI pattern**: `r2-ffi` is a `staticlib` with `cbindgen` generating `r2_ffi.h`. The `build.rs` reads `cbindgen.toml` for config.
- **Clippy**: Prefer `#[derive(Default)]` over manual impl when all fields have standard defaults. Clippy enforces this with `-D warnings`.
- **S3 client architecture**: Cloudflare REST API (token/accounts/buckets) uses `reqwest`; S3-compatible operations (objects/multipart) use `aws-sdk-s3`. Both live in `R2Client`. Static methods (`validate_token`, `list_accounts`) for pre-auth flows; instance methods for everything else.
- **R2 S3 credentials**: R2 uses the API token as both `access_key_id` and `secret_access_key`. Endpoint is `https://<account_id>.r2.cloudflarestorage.com`, region is `"auto"`, path-style forced.
- **Env var tests race condition**: Config tests that modify `R2DROP_HOME` env var can race when run in parallel. Don't assert exact equality on `config_dir()` return value ‚Äî another thread may have changed the env var.
- **Clippy div_ceil**: Rust 1.93+ clippy enforces `u64::div_ceil()` instead of manual `(a + b - 1) / b` pattern. Use the stdlib method.
- **Clippy too_many_arguments**: Private helper functions with >7 params need `#[allow(clippy::too_many_arguments)]`. Consider grouping params into structs for public APIs.
- **Async concurrency with buffer_unordered**: `futures::stream::buffer_unordered` runs N futures concurrently without requiring `'static` (unlike `tokio::spawn`). Ideal for parallel chunk uploads where you need to borrow from the enclosing scope. Use `async move` (not `async`) so closure params are moved into futures.
- **S3 custom metadata for dedup**: R2/S3 ETags are MD5-based, not SHA-256. To enable SHA-256 dedup, store the hash as custom metadata via `.metadata("sha256", hash)` on `put_object` / `create_multipart_upload`, then read it back from `head_object().metadata()`. The `hashes_match()` helper checks metadata first, falls back to ETag.
- **SHA-256 incremental hashing**: Use `sha2::Sha256` with a 64KB read buffer loop via `tokio::io::AsyncReadExt::read()`. No `hex` crate needed ‚Äî format bytes with `format!("{b:02x}")` and collect. Works for files > 4 GB.
- **Progress callback ownership**: When a function needs `progress_cb` and might fall back to a second call path (e.g. resume ‚Üí fresh upload), take `&Option<Box<dyn Fn(...)>>` not `Option<Box<...>>`. Ownership transfer prevents reuse.
- **Crash recovery state transitions**: Use existing state machine transitions (`Uploading ‚Üí Failed ‚Üí Pending`) instead of adding special-case shortcuts. Keeps validation logic centralized.
- **S3 list_parts for resume**: `aws-sdk-s3::list_parts` returns parts with `Option<i32>` part numbers. Use `unwrap_or(0)` to handle missing values.
- **C callback ‚Üí Swift AsyncStream**: C function pointers (`@convention(c)`) can't capture closures. Use module-level static storage (NSLock + optional `AsyncStream.Continuation`) to bridge. Non-capturing closures with `@convention(c)` annotation work as `R2ProgressCallback` values.
- **macOS 13 onChange**: `onChange(of:initial:_:)` with two-parameter closure requires macOS 14+. Use `onChange(of:) { newValue in }` for macOS 13 deployment target.
- **SwiftUI arbitrary windows**: `App` protocol can't open windows on launch. Use `NSApplicationDelegateAdaptor` + `NSWindow` + `NSHostingView` for onboarding/splash windows.
- **@MainActor AppDelegate**: `NSApplicationDelegate` runs on main thread but isn't implicitly `@MainActor`. If AppDelegate holds `@MainActor` services (like TokenValidationService), annotate AppDelegate with `@MainActor` to avoid concurrency errors.
- **StateObject with parameters**: Use `_viewModel = StateObject(wrappedValue: MyViewModel(param))` in SwiftUI view `init()` to pass parameters to `@StateObject` view models. The wrappedValue closure is only evaluated once on first appearance.
- **MenuBarExtra body is rebuilt on each open**: SwiftUI `MenuBarExtra` with `.menu` style evaluates `body` each time the menu opens. Safe to call `ConfigManager.load()` directly ‚Äî no need for `@State` or caching. Guarantees fresh account state.
- **NSAlert in menu bar apps**: `NSAlert.runModal()` works from menu bar app context for confirmation dialogs. Blocks the calling thread and returns which button was clicked.
- **NSStatusItem over MenuBarExtra**: SwiftUI's `MenuBarExtra` can't animate icons or accept drag-and-drop. Use `NSStatusItem` + `NSMenu` + `NSMenuDelegate.menuWillOpen` for full control. The delegate rebuilds the menu each time it opens for fresh data.
- **Drag-and-drop on NSStatusBarButton**: The button doesn't support `NSDraggingDestination` directly. Add a transparent `NSView` subview that registers for `.fileURL` drag types and forwards `mouseDown`/`mouseUp` to `superview` so clicks still open the menu. `hitTest` must NOT return nil (drag events are routed via view hierarchy).
- **SF Symbol template images for light/dark**: Set `image?.isTemplate = true` on `NSStatusItem.button.image`. macOS automatically renders the icon correctly in both light and dark menu bars. No hardcoded colors needed.
- **Icon animation via Timer**: Alternate between `arrow.up.circle.fill` and `arrow.up.circle` SF Symbols every 0.6s for a pulse effect. Timer fires on main run loop; wrap in `Task { @MainActor in }` for Swift concurrency safety.
- **SwiftUI App with only Settings scene**: When using NSStatusItem instead of MenuBarExtra, the App body can contain only a `Settings` scene. Works correctly for `LSUIElement = true` menu bar apps ‚Äî the NSStatusItem provides the visual presence.
- **Finder extension ‚Üí main app queue bridge**: Finder extension writes to App Groups shared container queue.db via `QueueManager(appGroup:)`. Main app polls this DB every 2s via `FinderQueueBridge`, transfers pending jobs to `~/.r2drop/queue.db`, and deletes from shared DB. Two separate SQLite files with the same schema.
- **FIFinderSync directory monitoring**: Use `URL(fileURLWithPath: "/")` to monitor the entire filesystem and make the "Send to R2" context menu available everywhere.
- **App Groups UserDefaults sharing**: `UserDefaults(suiteName: R2CoreConstants.appGroup)` shares preferences (like "Never ask again") between main app and Finder extension.
- **Finder extension stays FFI-free**: FinderExtension depends only on R2Core (models + data managers), not R2Bridge/Rust. Keeps the extension binary small and avoids linking libr2_ffi.a.
- **SMAppService for Launch at Login**: `SMAppService.mainApp` with `.register()/.unregister()` handles login items on macOS 13+. Much simpler than the old `SMLoginItemSetEnabled` API.
- **NSApp.setActivationPolicy for dock icon toggle**: `.accessory` hides the dock icon, `.regular` shows it. Can be called at runtime to toggle visibility.
- **Settings tab ViewModel pattern**: Load config in `onAppear`, each toggle/slider calls its setter which updates state + calls `save()` to persist immediately. No "Apply" button needed.
- **Task.detached for subprocess execution**: Use `Task.detached` (not `Task`) when running `Process` to avoid blocking the MainActor. Capture results in local vars, then `await MainActor.run { [weak self] in }` to update UI safely.
- **xcodebuild code signing override**: Local builds without a provisioning profile need four flags: `CODE_SIGN_IDENTITY=- CODE_SIGNING_REQUIRED=NO CODE_SIGNING_ALLOWED=NO PROVISIONING_PROFILE_SPECIFIER=""`. Missing `CODE_SIGNING_ALLOWED=NO` causes "requires a provisioning profile" errors.
- **Sparkle 2.x via SPM**: Add to XcodeGen `project.yml` under `packages:` with `url: https://github.com/sparkle-project/Sparkle` and `from: "2.6.0"`. Add `- package: Sparkle` to target dependencies. Use `SPUStandardUpdaterController(startingUpdater: true, ...)` for auto-update. Access `.updater` property for `checkForUpdates()`, `canCheckForUpdates`, `automaticallyChecksForUpdates`, and `lastUpdateCheckDate`.
- **AppKit import for NSWorkspace**: In pure SwiftUI files that need `NSWorkspace.shared.open(url)`, you must `import AppKit` (not just `Foundation`). SwiftUI's implicit AppKit availability doesn't cover `NSWorkspace`.
- **Deep link URL parsing**: `URL.host` gives the first component of `r2drop://host/path`. Use `URLComponents` for query params. Path segments come from `url.path`. For `r2drop://preferences/queue`, host is `"preferences"` and path is `"/queue"`.
- **SelectedTabStore singleton pattern**: Use `@MainActor final class` with `static let shared` and `@Published var requestedTab: SettingsTab?` to communicate tab selection from deep links to SettingsView. Consumer sets it to nil after reading.
- **NSApplicationDelegate URL handling**: Use `func application(_: NSApplication, open urls: [URL])` (not the older `application(_:open:)` single-URL variant). Receives an array of URLs.
- **UNUserNotificationCenter delegate on @MainActor**: The `UNUserNotificationCenterDelegate` methods must be marked `nonisolated` when the conforming class is `@MainActor`. Wrap async work in `Task { @MainActor in }` inside the delegate methods to get back to the main actor.
- **Notification categories with actions**: Register `UNNotificationCategory` with `UNNotificationAction` items via `setNotificationCategories()` during app launch. Set `content.categoryIdentifier` when posting to attach actions. Handle responses in `didReceive response:` delegate method using `response.actionIdentifier`.
- **Upload state transition detection**: Seed initial job IDs on service start to avoid false positives for pre-existing completed/failed jobs. Compare current poll against previous active IDs to detect transitions. Track notified IDs in sets to prevent duplicates.
- **NSSound.beep() for system sounds**: `NSSound.beep()` plays the system alert sound. No need to load a specific sound file. Works in menu bar apps.
- **Centralized notification service pattern**: Use a singleton `NotificationService.shared` that owns the `UNUserNotificationCenterDelegate` and all notification posting. Other services (like TokenValidationService) delegate to it instead of duplicating `UNUserNotificationCenter` code.
- **Synchronous FFI wrappers for Swift**: FFI functions using Rust's `runtime().block_on()` are inherently synchronous. Provide both async and sync Swift wrappers (`headObject` and `headObjectSync`) so callers don't need `DispatchSemaphore` hacks to call from non-async contexts.
- **Swift 5 try? flattens optionals**: `try? fn()` where `fn() throws -> T?` returns `T?` (not `T??`). In a `guard let x = try? fn()` chain, `x` is the unwrapped non-optional `T`. Don't double-unwrap with `let y = x` ‚Äî the compiler rejects it.
- **head_object for conflict detection**: `aws-sdk-s3` `HeadObjectOutput` provides `.content_length()` (returns `Option<i64>`) and `.last_modified()` (returns `Option<&DateTime>`). Use `.secs()` on DateTime to get epoch seconds for Swift interop.
- **NSAlert with three buttons for conflict dialog**: First button = `.alertFirstButtonReturn`, second = `.alertSecondButtonReturn`, third = default (anything else). Map to Overwrite/Skip/Rename. Add a checkbox accessory view for "Apply to all".
- **Session-scoped state via singleton**: For per-session preferences that reset on app restart (like conflict resolution "Apply to all"), use an `@MainActor` singleton with `static let shared`. No persistence needed ‚Äî instance dies with the process.
- **Conflict check at queue time, not upload time**: Check for conflicts via `head_object` before inserting into `queue.db`. This avoids modifying the Rust runner flow or adding new queue statuses. Both entry points (drag-drop and Finder extension bridge) perform the check.
- **Tracing for audit logging**: Use `tracing` + `tracing-subscriber` + `tracing-appender` for structured logging. `tracing-appender::rolling::Builder` supports `max_log_files()` for retention. Daily rotation via `Rotation::DAILY`. `try_init().ok()` makes init idempotent (safe in tests). FFI exposes `r2_init_logging(max_files)` so Swift calls once at startup.
- **Clippy io_other_error**: Rust 1.93+ clippy enforces `std::io::Error::other(e)` instead of `std::io::Error::new(ErrorKind::Other, e)`. Use the shorthand.
- **Cross-platform keyring via `keyring` crate**: Use `keyring::Entry::new(SERVICE, account)` with service `"com.superhumancorp.r2drop"` to match Swift's `KeychainManager`. On macOS, `keyring` uses Security.framework so entries are interoperable with the Swift app. `Entry::new` returns `Result` in keyring 3.x.
- **rpassword version**: Use `rpassword = "7"` (not 5). The `prompt_password()` function was introduced in 7.x. Version 5.x only has `read_password_from_tty()`.
- **Account struct has account_id**: `Account` in config.rs includes `account_id: Option<String>` with `#[serde(default)]` for backward compatibility. Needed for API calls. Matches US-009 spec for Swift Account model.
- **CLI standalone upload pattern**: Use `upload::upload_file()` directly for CLI uploads rather than going through queue.db + runner. The queue/runner loop is for the Swift app's background processing. CLI uploads are synchronous (user watches terminal).
- **zip crate v2 API**: `zip::write::SimpleFileOptions::default().compression_method(zip::CompressionMethod::Deflated)` for options. `ZipWriter::add_directory(name, opts)` for dirs, `start_file(name, opts)` + `io::copy` for files. `finish()` returns `Result<W, ZipError>`.
- **Terminal progress with \\r overwrite**: `print!("\\r    {pct}%  {speed}  {eta}          ")` + `stdout().flush()` overwrites the current line. Trailing spaces clear leftover chars. Use `print!("\\r    done                              \\n")` to replace progress with final result.
- **Clap nested subcommands**: Use `#[command(subcommand)]` with a `clap::Subcommand`-derived enum for nested subcommands (e.g. `r2drop config get/set`). The parent variant wraps the enum directly, not via `Args`.
- **CLI --json flag pattern**: Add `#[arg(long)] json: bool` to each command's args struct. Use `serde_json::json!({...})` + `to_string_pretty()` for JSON output. Keeps human-readable as default.
- **CLI module naming**: Avoid module names that shadow common crate names. Use suffixed names like `config_cmd` and `history_cmd` instead of `config` and `history`.
- **CI Xcode build without signing**: Use `CODE_SIGN_IDENTITY=- CODE_SIGNING_REQUIRED=NO CODE_SIGNING_ALLOWED=NO PROVISIONING_PROFILE_SPECIFIER=""` with `xcodebuild`. The Rust static library must be pre-built and copied to `engine/target/release/` before `xcodebuild build` runs.
- **CI keychain for code signing**: Create a temp keychain (`security create-keychain`), import the .p12 cert, set partition list (`-S apple-tool:,apple:`) so `codesign` can access without prompts, and add to search list. Always clean up in an `if: always()` step.
- **Notarization pipeline**: ZIP with `ditto -c -k --keepParent`, submit via `xcrun notarytool submit --wait --timeout 1800`, then `xcrun stapler staple`. Both .app and .dmg need independent notarization.
- **CLI cross-compile matrix**: Linux ARM64 needs `gcc-aarch64-linux-gnu` + `.cargo/config.toml` linker override. macOS and Windows targets work natively. Use `strategy.fail-fast: false` so one platform failure doesn't cancel others.
- **Release concurrency**: Use `cancel-in-progress: false` for release workflows ‚Äî never abort a release mid-flight.
- **AppKit reduced motion check**: AppKit Timer-based animations can't use SwiftUI's `@Environment(\.accessibilityReduceMotion)`. Use `NSWorkspace.shared.accessibilityDisplayShouldReduceMotion` instead. Check on each timer frame for dynamic toggling support.

  - For crash recovery, transitioning through `Uploading ‚Üí Failed ‚Üí Pending` reuses existing state machine validation rather than adding a special `Uploading ‚Üí Pending` shortcut.
- **cbindgen type alias export**: cbindgen only emits type aliases used by exported functions. To force-export a standalone type like `R2ProgressCallback`, add `[export] include = ["TypeName"]` to `cbindgen.toml`.
- **FFI type aliases for cbindgen**: Use `extern "C" fn(...)` (not `Option<extern "C" fn(...)>`) for type aliases exported to C. `Option<extern "C" fn>` is Rust-specific; C handles nullable function pointers natively.
- **FFI async runtime**: Use `OnceLock<tokio::runtime::Runtime>` for a lazily-initialized shared runtime. Call `runtime().block_on(future)` from synchronous `extern "C"` functions.
- **FFI error pattern**: Thread-local `RefCell<Option<String>>` for `LAST_ERROR`. Functions return status codes (0/-1), caller reads error via `r2_get_last_error()`. Thread-local avoids locking and concurrent-call clobbering.
- **FFI string ownership**: Rust allocates strings via `CString::into_raw()`, Swift frees them via `r2_free_string()` which calls `CString::from_raw()`. C strings passed into FFI functions are borrowed (not freed by Rust).
- **XcodeGen for project generation**: Use `xcodegen` (brew install) to generate `.xcodeproj` from `project.yml`. Avoids hand-writing pbxproj. Run `cd R2Drop && xcodegen generate` after any project structure changes.
- **XcodeGen Finder Sync Extension type**: XcodeGen doesn't have a `finder-sync-extension` type. Use `app-extension` and configure `NSExtension.NSExtensionPointIdentifier = "com.apple.FinderSync"` in the Info.plist properties.
- **XcodeGen entitlements**: Specifying `entitlements.path` without `entitlements.properties` generates an empty entitlements plist. Always include `properties` to populate entitlement keys.
- **R2Bridge C module map**: SPM wraps the Rust FFI via a `R2BridgeC` target with `module.modulemap` importing `r2_ffi.h`. The actual `libr2_ffi.a` is linked at the Xcode project level via `OTHER_LDFLAGS` and `LIBRARY_SEARCH_PATHS`, not SPM.
- **Rust static lib system frameworks**: Linking `libr2_ffi.a` requires `-framework Security -framework SystemConfiguration -framework CoreFoundation` in `OTHER_LDFLAGS` because Rust deps (TLS, network) use these macOS frameworks.
- **xcodebuild CODE_SIGNING_ALLOWED=NO**: For CI/dev builds without signing certificates, pass `CODE_SIGNING_ALLOWED=NO` to xcodebuild. This skips codesigning while still validating the build.
