Best Cloudflare R2 Upload Tools for macOS in 2026

Tools
—
March 1, 2026

Cloudflare R2 is one of the most cost-effective object storage services available, thanks to zero egress fees and an S3-compatible API. But getting files into R2 is not as obvious as it should be. Unlike AWS S3, which has years of tooling built around it, R2 is still relatively new. If you're on macOS and looking for the best way to upload files to Cloudflare R2, this guide compares every major option available in 2026.

We'll look at seven tools: the Cloudflare dashboard, Wrangler CLI, rclone, Cyberduck, Transmit, boto3/AWS SDK, and R2Drop. Each has different strengths depending on whether you need a GUI, terminal workflow, scripting capability, or just the fastest path from file to bucket.

1. Cloudflare Dashboard (Web UI)

The most obvious option. Log into dash.cloudflare.com, navigate to R2, select your bucket, and click upload. No installation required.

Strengths: Zero setup. Works on any OS with a browser. Good for quick one-off uploads when you're already in the dashboard. Supports creating folders and viewing existing objects.

Weaknesses: Painfully slow for repeated uploads. You need to navigate through multiple screens every time. No drag-and-drop from Finder. File size limits apply (the dashboard caps uploads at 300 MB per file). No multipart upload support, no resume on failure, and no URL copied to clipboard automatically. If you upload more than a handful of files per week, this gets old fast.

Best for: One-off uploads, viewing bucket contents, managing bucket settings.

2. Wrangler CLI

Wrangler is Cloudflare's official CLI tool, primarily built for Workers development. It includes wrangler r2 object put and wrangler r2 object get commands for R2 operations.

wrangler r2 object put my-bucket/path/to/file.png --file ./file.png

Strengths: Official Cloudflare tool, well-maintained, good documentation. Supports piping from stdin. Integrates with your existing Cloudflare auth (OAuth via wrangler login). Works well in CI/CD pipelines alongside Workers deployments.

Weaknesses: Requires Node.js and npm. The r2 object put command only uploads one file at a time — there's no recursive folder upload built in. No multipart upload for large files. The auth model is tied to Cloudflare's OAuth, which can be awkward for service accounts. Wrangler is a large dependency if you only need R2 uploads.

Best for: Developers already using Wrangler for Cloudflare Workers who need occasional R2 uploads.

3. rclone

rclone is a mature, open-source command-line tool for syncing files to cloud storage. It supports over 70 backends, including S3-compatible services like Cloudflare R2.

rclone copy ./local-folder r2remote:my-bucket/path/ --progress

Strengths: Extremely powerful. Supports sync, copy, move, mount, and dozens of other operations. Handles large files with multipart uploads. Parallel transfers are configurable. Extensive filtering options (include/exclude patterns). Battle-tested with years of community use. Free and open source.

Weaknesses: Configuration is not trivial. You need to run rclone config and manually enter your R2 endpoint, access key, and secret key. The config file stores credentials in plaintext at ~/.config/rclone/rclone.conf unless you enable encryption. No GUI. Not macOS-native — no Finder integration, no menu bar, no system notifications. The learning curve is real if you've never used it before.

Best for: Power users who need sync, mount, or complex transfer operations. Bulk migrations from other cloud providers to R2.

4. Cyberduck

Cyberduck is a free, open-source GUI file transfer client for macOS (and Windows). It supports S3-compatible storage, including Cloudflare R2, through connection profiles.

Strengths: Familiar FTP-client-style interface. Free (donation-supported). Drag-and-drop file uploads. Bookmark connections for quick access. Supports browsing bucket contents. Available on the Mac App Store.

Weaknesses: The R2 connection requires a custom connection profile or manual S3-compatible configuration — it doesn't have a dedicated "Cloudflare R2" option out of the box. The UI feels dated compared to modern macOS apps. Upload speeds are decent but not optimized for R2 specifically. No Finder context menu integration. No automatic URL copying. It's a general-purpose transfer client, not purpose-built for R2.

Best for: Users who want a free GUI for browsing and managing R2 bucket contents, not just uploading.

5. Transmit (by Panic)

Transmit is a premium macOS file transfer app from Panic, the makers of Nova. It supports S3-compatible storage and has a polished, native macOS interface.

Strengths: Beautiful, native macOS design. Fast multi-connection transfers. Supports S3-compatible endpoints including R2. Finder integration via the Transmit Disk feature (mounts remote storage as a local volume). Sync features for comparing local and remote directories.

Weaknesses: Costs $45 (one-time purchase). The S3 configuration requires manual endpoint entry for R2. Transmit Disk can be flaky with S3-compatible services that aren't actual AWS S3. It's a general-purpose file transfer app — you're paying for FTP, SFTP, WebDAV, and other protocols you may not need. No automatic public URL generation after upload.

Best for: Developers who already own Transmit and want a dual-pane file manager for R2. Users who need to browse, download, and manage bucket contents alongside uploads.

6. boto3 / AWS SDK

Because Cloudflare R2 is S3-compatible, you can use the AWS SDK (boto3 for Python, @aws-sdk/client-s3 for Node.js, etc.) to interact with R2 programmatically.

import boto3

s3 = boto3.client('s3',
    endpoint_url='https://ACCOUNT_ID.r2.cloudflarestorage.com',
    aws_access_key_id='YOUR_ACCESS_KEY',
    aws_secret_access_key='YOUR_SECRET_KEY'
)
s3.upload_file('local-file.png', 'my-bucket', 'remote/path/file.png')

Strengths: Maximum flexibility. Full programmatic control over uploads, metadata, content types, and ACLs. Supports multipart uploads natively. Ideal for integration into existing Python/Node.js/Go applications. Well-documented (AWS S3 docs apply almost entirely).

Weaknesses: Requires writing code. Not a "tool" in the traditional sense — it's a library. You need to manage credentials yourself (environment variables, config files, etc.). No GUI, no Finder integration, no macOS-native features. Overkill for simple file uploads.

Best for: Application-level integration. Automated upload pipelines. Custom tooling built on top of R2.

7. R2Drop

R2Drop is a native macOS menu bar app built specifically for uploading files to Cloudflare R2. Right-click any file or folder in Finder, select "Send to R2," and the file is uploaded in the background. The URL is automatically copied to your clipboard.

Strengths: Purpose-built for R2 on macOS. Finder right-click integration (no browser or terminal needed). Automatic multipart uploads with parallel chunks for large files. Resumable uploads. Credentials stored in macOS Keychain (not plaintext config files). Multiple account support. Automatic URL copying to clipboard. Upload history with search. Includes a CLI (r2) for terminal and CI/CD use. Free and open source (MIT). Rust-powered upload engine for performance.

Weaknesses: macOS only — no Windows or Linux support. Upload-only: you cannot browse or download bucket contents. Relatively new project (launched February 2026). No sync or mount capability.

Best for: macOS developers who upload files to R2 regularly and want the fastest, most native experience. Anyone who wants Finder integration and automatic URL copying without touching a terminal.

Which Tool Should You Use?

There's no single "best" tool for everyone. The right choice depends on your workflow.

If you're a macOS developer who uploads to R2 regularly — screenshots, build artifacts, images, static files — R2Drop is the most frictionless option. Right-click, upload, URL on clipboard. No browser tabs, no terminal commands. It also includes a CLI for when you do want terminal access.

If you need to sync large directories or migrate terabytes of data, rclone is unbeatable. Its sync engine is battle-tested across dozens of cloud storage providers and it handles edge cases (interrupted transfers, delta sync, bandwidth throttling) that no other tool on this list matches.

If you're building an application that needs to upload to R2 programmatically, use the AWS SDK (boto3, @aws-sdk/client-s3, etc.). You get full control over every aspect of the upload.

If you want to browse your R2 bucket contents with a GUI — viewing, downloading, and organizing files — Cyberduck or Transmit are the right fit. R2Drop is upload-only by design.

If you're already deep in the Cloudflare Workers ecosystem and just need to push a file occasionally, Wrangler works fine. It's already on your machine.

And if you just need to upload one file right now and don't want to install anything, the Cloudflare dashboard is always there.

Getting Started with R2Drop

If R2Drop sounds like the right fit for your workflow, getting started takes about 60 seconds. Install it via Homebrew (brew install --cask r2drop) or download the .dmg from GitHub Releases. Add your Cloudflare R2 API token, and you're uploading from Finder immediately. For a full walkthrough, see our Getting Started guide.
